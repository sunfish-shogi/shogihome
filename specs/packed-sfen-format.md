# Packed SFEN 仕様（YaneuraOu 準拠）

この文書は、局面圧縮データ `PackedSfen`（やねうら王系）そのものの仕様を定義する。  
`PackedSfenValue`（評価値・指し手・手数・勝敗を含むレコード）は対象外とする。

## 1. データサイズ

- `PackedSfen` は **固定長 32 byte（256 bit）**。

## 2. ビットストリーム規約

- ビットは **LSB-first** で詰める。
- 1bit 書き込みは `data[bit_cursor / 8]` の `(bit_cursor & 7)` ビット目に書く。
- 複数ビット値は下位ビットから順に出力する（little-endian bit order）。

## 3. 全体レイアウト

`PackedSfen` のビット列は次の順で連結する。

1. 手番: 1 bit (`0=先手`, `1=後手`)
2. 先手玉位置: 7 bit
3. 後手玉位置: 7 bit
4. 盤上の非玉 79 マス: 可変長ハフマン
5. 手駒 + 駒箱: 可変長ハフマン（終端は `bit_cursor == 256`）

補足:

- 詰将棋などで片方の玉が存在しない場合、玉位置に `81`（`SQ_NB`）を格納する。

## 4. 盤上/手駒で使うハフマン基底表

以下はやねうら王 `extra/sfen_packer.cpp` 準拠の基底コード。

| 駒種       | code (hex) | bits |
| ---------- | ---------- | ---- |
| `NO_PIECE` | `0x00`     | 1    |
| `PAWN`     | `0x01`     | 2    |
| `LANCE`    | `0x03`     | 4    |
| `KNIGHT`   | `0x0b`     | 4    |
| `SILVER`   | `0x07`     | 4    |
| `GOLD`     | `0x0f`     | 5    |
| `BISHOP`   | `0x1f`     | 6    |
| `ROOK`     | `0x3f`     | 6    |

## 5. 駒箱専用ハフマン表（駒落ち対応）

駒箱（盤上にも手駒にもない駒）を書き出すときは以下を使う。

| 駒種     | code (hex) | bits |
| -------- | ---------- | ---- |
| `PAWN`   | `0x02`     | 2    |
| `LANCE`  | `0x09`     | 4    |
| `KNIGHT` | `0x0d`     | 4    |
| `SILVER` | `0x0b`     | 4    |
| `GOLD`   | `0x1b`     | 5    |
| `BISHOP` | `0x2f`     | 6    |
| `ROOK`   | `0x3f`     | 6    |

## 6. 各要素のエンコード

### 6.1 盤上の駒（玉以外）

1. 基底表の `code/bits` を出力。
2. `NO_PIECE` なら終了。
3. 駒種が `GOLD` 以外なら成りフラグ 1 bit（`0/1`）を出力。
4. 先後フラグ 1 bit（`0=先手`, `1=後手`）を出力。

### 6.2 手駒

1. 基底表の `code` から LSB 1bit を除いた値 (`code >> 1`, `bits - 1`) を出力。
2. 駒種が `GOLD` 以外なら成りフラグ 1 bit を出力（手駒では通常 `0`）。
3. 先後フラグ 1 bit を出力。

### 6.3 駒箱

1. 駒箱専用表で `code/bits` を出力。
2. 駒種が `GOLD` 以外なら `0` を 1 bit 追加出力。

補足:

- 復号側は「手駒デコーダ」で最後まで読み、**成りフラグ付きとして解釈された駒を駒箱として捨てる**実装で整合する。
- `GOLD` の駒箱表現はこの互換仕様（cshogi 互換）に合わせる。

## 7. 駒の列挙順

### 7.1 盤上79マス

- 81 マスを走査し、玉が置かれている 2 マスをスキップして出力する。
- 走査順はやねうら王の `Square` 列挙順に一致させること。

### 7.2 手駒と駒箱

色順:

- 先手 → 後手

駒種順:

- `PAWN`, `LANCE`, `KNIGHT`, `SILVER`, `GOLD`, `BISHOP`, `ROOK`

駒箱枚数の初期値（総駒数）:

- `PAWN=18`, `LANCE=4`, `KNIGHT=4`, `SILVER=4`, `GOLD=4`, `BISHOP=2`, `ROOK=2`

`駒箱枚数 = 総駒数 - 盤上枚数 - 手駒枚数` を同順で出力する。

## 8. 復号

1. 先頭 15 bit で手番と玉2枚を復元。
2. 81 マスを順に見て、玉マス以外は盤上駒を 1 枚ずつ復号。
3. 残りビットは 256 bit 到達まで「手駒デコーダ」で読取り。
4. 成り駒として読まれたものは駒箱として破棄し、非成りのみ手駒カウントに加算。

## 9. 256 bit 固定長の性質

- 通常局面（40駒）でも駒落ち局面でも、駒箱エンコードを含めることで常に 256 bit になる。
- したがって、`PackedSfen` はストリーム上で 32 byte 固定長として扱える。
