# .sbk 定跡フォーマット仕様（フォーマット定義）

この文書は `.sbk` ファイルの**データフォーマットそのもの**を定義する。  
アプリケーション固有の運用フローや UI 挙動は対象外とする。

## 1. ファイル全体

- `.sbk` は Protocol Buffers のバイナリで、トップレベルは `SBook` 1 メッセージ。
- ファイルヘッダ（magic number / バージョン / 圧縮種別）は持たない。
- 文字列は protobuf の通常仕様に従い UTF-8。
- 不明フィールドは protobuf の通常仕様どおり無視可能。

## 2. ノードグラフ表現

- `SBookState` が局面ノード。
- `SBookMove.NextStateId` が遷移先ノード参照。
- `NextStateId = -1` は「遷移先ノードなし（終端）」を表す値として使用可能。

### 2.1 ID と配列インデックス

- 互換性のため、`BookStates[i].Id == i`（0 始まり連番）を推奨。
- `NextStateId` はこのインデックス（= Id）を参照する前提で扱う。
- 実装互換性を重視する場合、`Position` を持つルートノードを先に並べる。

### 2.2 ルートノードと Position

- `Position`（SFEN 文字列）を保持する `SBookState` をルートノードとみなす。
- 参照される中間ノード/子ノードでは `Position` を省略可能（空文字/未設定）。
- `Position` を保持する場合の文字列形式は SFEN（`board side hand moveCount`）。
- `moveCount` は実運用上 `1` が使われることが多い。

## 3. 数値フィールドの扱い

- `BoardKey` / `HandKey` は 0 を許容する。
- `Games` / `WonBlack` / `WonWhite` / `Weight` / 評価関連数値は 0 を許容する。
- protobuf エンコーダ実装により、0 や空文字はワイヤに出力されない場合がある。  
  したがってデコーダは「未出力 = 既定値（0 / 空）」として扱える必要がある。

## 4. `SBookMove.Move` の 32bit レイアウト

`Move` は `int32`（2 の補数）として格納されるビットフィールド。

| bit   | 意味       | 備考                             |
| ----- | ---------- | -------------------------------- |
| 0-3   | `fromDan`  | 1-9。打ち手は 0                  |
| 4-7   | `fromSuji` | 1-9。打ち手は 0                  |
| 8-11  | `toDan`    | 1-9                              |
| 12-15 | `toSuji`   | 1-9                              |
| 16-18 | 予約       | 0 を推奨                         |
| 19    | 成りフラグ | 1=成り                           |
| 20-23 | 取得駒種   | 駒種インデックス（成り情報含む） |
| 24-28 | 移動駒種   | 駒種インデックス（5bit）         |
| 29-30 | 予約       | 0 を推奨                         |
| 31    | 手番       | 0=先手, 1=後手                   |

### 4.1 判定ルール

- 打ち手: `fromDan == 0 && fromSuji == 0`
- 手番: bit31 を参照（`int32` としては `Move < 0` が後手）
- `CapturePiece` は駒種のみを保持し、先後はこのフィールドには含めない。

### 4.2 駒種インデックス

駒種は BookConv 系の `Piece` 列挙値に基づく。

- `0`: なし
- `1`: 歩
- `2`: 香
- `3`: 桂
- `4`: 銀
- `5`: 金
- `6`: 角
- `7`: 飛
- `8`: 玉
- `9`: と
- `10`: 成香
- `11`: 成桂
- `12`: 成銀
- `14`: 馬
- `15`: 龍

補足:

- `13` は未使用。
- 成駒は元駒インデックスに成りビットを加えた値で表現される。
- 一部実装は駒種フィールド内に先後ビット（`0x10`）を持つが、手番判定は bit31 を優先する。

## 5. `Evaluation` 列挙値

- `0`: None
- `1`: Forced
- `2`: Good
- `3`: Bad
- `4`: Blunder

## 6. 実装互換性チェックリスト（推奨）

- `BookStates` は 0 始まり連番 ID と同順序で配置する。
- `NextStateId` は有効範囲内の ID か `-1` を使う。
- ルート局面のみ `Position` を保持し、その他は省略してよい。
- `Move` の予約ビット（16-18, 29-30）は 0 にする。
